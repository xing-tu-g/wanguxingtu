# Hermes 会话规则：万古星图

目标：让 Hermes 作为项目规划/调度/验收控制器，避免长会话 token 膨胀。

## 上下文预算

- 不要整本读取 `docs/HANDOFF.md`；续接时只读末尾 80-140 行，或用搜索读取与当前里程碑相关的小段。
- 续接优先读 `docs/CURRENT.md`，再读 `docs/HANDOFF.md` 末尾 80-140 行；禁止整本读取 `docs/HANDOFF.md`。
- 不要把完整 Godot 回归日志、完整 Codex 输出、完整大文件内容粘进回复；只保留命令、结论、失败摘要和产物路径。
- 读取文件优先使用小范围 `offset` / `limit`；需要全局了解时先搜索文件名或关键词。
- 若会话开始接近上下文上限，先 `/compress` 或开新会话，不要继续堆积工具输出。

## 工作模式

- Hermes 主会话负责：确定里程碑、给 Codex 下窄任务、独立验证、更新交接、给用户中文摘要。
- 实现型任务优先委托 Codex CLI；Hermes 不在主会话里反复读取/粘贴大段代码。
- Codex 任务必须窄而自包含，要求返回：改动文件、验证命令、真实结果、阻塞项。
- Hermes 对 Codex 结果只采信可验证事实；必须自己运行或读取验证产物后再汇报。

## Windows / Godot

- 项目根目录是 `D:\wanguxingtu` / `/d/wanguxingtu`。
- 使用 `godot.cmd`，不要使用 extensionless `godot`。
- 使用 `C:\Users\23503\bin\codex.cmd` 或 `/c/Users/23503/bin/codex.sh` 启动 Codex。
- 玩家可见 UI、日志、交接摘要使用中文。

## 交接

- 每个阶段完成后更新 `docs/HANDOFF.md`，但内容要摘要化：状态、验证命令、结果、产物路径、阻塞项、下一步。
- 每个阶段完成后先更新 `docs/CURRENT.md`，再追加摘要到 `docs/HANDOFF.md`；内容要摘要化：状态、验证命令、结果、产物路径、阻塞项、下一步。
- 不要把临时长日志写入交接；长日志只记录文件路径。
