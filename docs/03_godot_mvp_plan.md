# 万古星图：Godot MVP 开发清单

版本：v0.1-MVP  
目标：用 Hermes + Codex + Godot 从 0 到 1 打通核心玩法。

---

## 1. MVP 目标

MVP 不追求完整商业闭环，只验证核心战斗是否成立。

必须达成：

1. 玩家能进入战斗。
2. 棋盘为 10×5。
3. 双方有奕星师 HP。
4. 玩家能在己方前三列部署武将。
5. 单位能自动移动、自动攻击。
6. 能触发基础技能、地形、星力、星潮。
7. 能结束战斗并展示本场统计。
8. 安卓横屏 UI 可操作。

暂不开发：

- 真实抽卡。
- 充值。
- 联网排位。
- 完整背包。
- 完整 42 武将。
- MVP 武将评选。

---

## 2. 推荐 Godot 技术选型

- 引擎：Godot 4.x。
- 语言：GDScript。
- 画面：2D 横屏。
- 数据：JSON 或 Godot Resource，MVP 推荐 JSON，方便 Codex 生成和修改。
- 存档：本地 `user://save.json`。
- 平台优先级：Windows 编辑器调试 → Android 横屏真机。

## 4. 开发里程碑

### M0：工程初始化

交付物：Godot 空工程可运行。

任务：

- 创建 Godot 4 项目。
- 设置横屏 Android 配置。
- 创建 `Boot` 场景。
- 创建基础页面路由。
- 加载 JSON 数据。

验收：

- Windows 编辑器可运行。
- 启动后进入首页。
- 首页可跳转到战前准备和对战页占位。

---

### M1：棋盘与部署

交付物：可在 10×5 棋盘部署单位。

任务：

- 实现棋盘模型。
- 实现 10×5 Tile UI。
- 标记己方部署区、公共区域、敌方部署区。
- 实现部署合法性判断。
- 实现星力消耗。
- 实现单位实例生成。

验收：

- 玩家只能在己方前三列部署。
- 公共四列不能直接部署。
- 已占用格不能部署。
- 星力不足不能部署。

---

### M2：自动移动与攻击

交付物：单位能自动推进并攻击。

任务：

- 实现单位移动值。
- 实现只向前移动的简化推进逻辑。
- 实现阻挡判断；MVP 不做绕路。
- 实现曼哈顿攻击范围判断。
- 实现目标优先级：优先攻击距离我方阵地最近的单位。
- 实现单位死亡和移除。
- 实现奕星师受伤。

验收：

- 点击“推进回合”后，当前方单位自动移动和攻击。
- 单位会优先攻击范围内靠近我方阵地的敌人。
- 无敌方单位可打时能攻击敌方奕星师。
- 一方奕星师 HP <= 0 时进入结算。

---

### M3：回合制流程与星潮

交付物：完整先后手循环。

任务：

- 实现先后手。
- 实现先手 5 星力、后手 6 星力。
- 实现每回合恢复星力。
- 实现星力上限 10。
- 实现星潮恢复量增加。
- 实现第 8 / 12 / 16 回合起对奕星师伤害加成。

验收：

- 回合顺序清晰显示。
- 星力变化正确。
- 星潮开始后恢复量增加。
- 第 8 / 12 / 16 回合起，对奕星师伤害加成正确。

---

### M4：地形与策略卡

交付物：地形和策略卡能影响战斗。

任务：

- 开局随机生成 5 个特殊地形。
- 实现草地、泥沼、河流、高地。
- 实现策略卡数据和抽取。
- 实现鼓舞、补给、急行军。
- 后续再实现火矢、落石、地震。

验收：

- 每局地形分布不同。
- 高地能增加射手 / 法师射程。
- 河流能影响攻击与受伤。
- 补给回复奕星师 10 HP。
- 地震造成双方奕星师 10 伤害。

---

### M5：12 武将技能

交付物：MVP 12 武将可用。

优先实现：

1. 关羽：回合开始成长。
2. 诸葛亮：部署时范围生命增益。
3. 赵云：移动穿越。
4. 张飞：邻位保护。
5. 曹操：全军推进。
6. 司马懿：部署沉默。
7. 张辽：击杀再行动。
8. 周瑜：燃烧。
9. 孙尚香：追加射击。
10. 陆逊：燃烧目标增伤。
11. 吕布：击杀再行动。
12. 张角：召唤黄巾兵。

验收：

- 每名武将至少有一个可观察的技能表现。
- 技能触发写入战斗日志。
- 技能数值来自数据表。

---

### M6：页面闭环

交付物：首页 → 战前准备 → 对战 → 结算 → 首页闭环。

任务：

- 首页：进入武将、战前准备、对战。
- 武将页：展示 MVP 武将列表和基础信息。
- 战前准备：选择 15 人阵容；MVP 可用预设阵容。
- 对战页：棋盘、HUD、动作按钮、日志。
- 结算页：胜负和本场统计。

验收：

- 不通过编辑器调试菜单也能完成一局。
- 结算后能返回首页或再来一局。

---

### M7：安卓横屏可玩包

交付物：安卓 APK 可安装、可横屏操作、可完成一局。

任务：

- 横屏强制。
- 触控按钮尺寸检查。
- 安全区适配。
- 字体和 UI 缩放适配。
- 构建 Android APK。
- 真机基础验证。

验收：

- 安卓手机横屏启动。
- 主要按钮不被刘海 / 圆角 / 导航栏遮挡。
- 棋盘和 HUD 在常见屏幕比例下可读。
- 能完成一局战斗。

| 页面 | 优先级 | 说明 |
|---|---|---|

## 6. 安卓横屏 UI 规范

### 6.1 设计基准

- 设计基准分辨率：1920×1080 横屏。
- 最小适配目标：1280×720 横屏。
- 高比例适配：20:9、21:9 横屏手机。
- UI 使用锚点和比例布局，不使用 Word 文档里的 cm / pt 作为实现单位。

### 6.2 安全区

- 左右保留安全边距：至少 32 px。
- 顶部保留安全边距：至少 24 px。
- 底部保留安全边距：至少 32 px。
- 对有刘海、挖孔、圆角、手势条的设备，核心按钮不得贴边。

### 6.3 触控尺寸

- 主要按钮最小高度：56 px。
- 次级按钮最小高度：48 px。
- 可点击卡牌最小触控区域：72×72 px。
- 棋盘格点击区域不得小于 64×64 px；小屏下优先缩小边栏，不缩小棋盘到不可点。

### 6.4 对战页布局

推荐横屏结构：

```text
┌──────────────────────────────────────────────┐
│ 顶部 HUD：回合 / 星力 / 双方奕星师 HP / 设置 │
├───────────────┬──────────────────────┬───────┤
│ 我方手牌/候选 │      10×5 棋盘        │ 日志  │
│ 或部署按钮    │                      │ 可收起│
├───────────────┴──────────────────────┴───────┤
│ 底部操作区：自动布我方 / 推进回合 / 详情提示 │
└──────────────────────────────────────────────┘
```

小屏策略：

- 日志默认收起，用按钮展开。
- 单位详情使用弹层，不常驻占屏。
- 候选武将可横向滚动。
- 棋盘始终居中优先显示。

### 6.5 字号建议

以 1920×1080 横屏为基准：

| 用途 | 推荐字号 |
|---|---:|
| 页面标题 | 32–40 px |
| 模块标题 | 24–28 px |
| 正文 | 20–24 px |
| 数值 / 标签 | 18–22 px |
| 战斗日志 | 16–20 px |

小屏可按比例缩放，但正文不应低于 16 px。

### 6.6 色彩与风格

- 主风格：Q 版萌系 + 亮色蓝白主题。
- 传说：金色。
- 史诗：紫色。
- 精英：蓝色。
- 危险 / 伤害：红色。
- 治疗 / 增益：绿色。
- 星力 / 星潮：蓝紫色或星光色。

### 6.7 信息优先级

对战页最高优先级信息：

1. 双方奕星师 HP。
2. 当前回合与行动方。
3. 当前星力。
4. 棋盘单位位置。
5. 可部署武将。
6. 战斗日志。

## 7. Codex 执行建议

每次交给 Codex 的任务应小而明确。

推荐任务粒度：

- “创建 Godot 项目结构和数据加载器”。
- “实现 10×5 BoardModel 和部署合法性测试”。
- “实现 TargetingSystem，按距离我方阵地最近选择目标”。
- “实现星力和回合循环”。
- “实现 3 张策略卡：鼓舞、补给、急行军”。
- “实现关羽、周瑜、张角三个技能作为技能系统样板”。

避免一次性要求：

- “把整个游戏做完”。
- “实现全部 42 武将”。
- “完整上线版安卓手游”。

---

## 8. 开发验收顺序

1. 编辑器内完成一局。
2. 桌面导出可完成一局。
3. Android 横屏可完成一局。
4. 加入更多武将。
5. 加入更多页面。
6. 调整数值和节奏。
7. 再考虑抽卡、经济、排位。

---

## 9. 项目规则文件建议

在 Godot 项目根目录创建 `AGENTS.md`，供 Hermes / Codex 读取。

建议内容：

```markdown
# 万古星图 Godot 项目规则

- 使用 Godot 4.x 和 GDScript。
- 优先实现核心战斗，不提前开发充值、联网、完整抽卡。
- 数据来自 `res://data/*.json`，不要把数值硬编码在脚本里。
- 棋盘规则以 `docs/01_rules_spec.md` 为准。
- 数值和武将池以 `docs/02_values_and_content.md` 为准。
- 安卓横屏适配必须保留安全区和触控尺寸。
- 每次修改后至少运行 Godot 编辑器内基础验证或脚本测试。
```
