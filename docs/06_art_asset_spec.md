# 万古星图：Q版星图战棋美术资产规格

版本：v0.2-Q版战斗表现方向  
目标：把美术方向从“漂亮星空背景 + 灰盒棋盘”调整为“英气 Q 版武将小人 + 星图棋盘 + 技能特效反馈”的战斗表现体系，方便后续交给图像专家继续优化。

> 当前约束：本文件只定义规格，不调用图片生成；正式产图前仍由用户/图像专家确认风格样张。

---

## 1. 核心美术决议

《万古星图》的战斗不是静态卡牌展示，而是武将会在 10×5 棋盘上移动、攻击、释放技能、受击和退场的自动战棋。因此美术主角必须从“背景”转为“可动武将 + 技能特效 + 命中反馈”。

当前推荐统一方向：

> **英气 Q 版星图战棋风**  
> 小比例武将，但不幼稚；动作夸张，但轮廓清楚；背景克制，棋盘清晰，特效负责爽感。

### 1.1 为什么不采用头像棋子作为最终方向

- 头像难表现挥刀、冲锋、施法、受击、死亡等动作。
- 方向感弱，不利于表现左右推进和攻击路径。
- 技能只能像 UI 特效贴在头像上，缺少“武将在战斗”的感觉。
- 可以作为早期临时占位，但不作为最终战斗表现载体。

### 1.2 Q版的边界

允许：

- 2.5 头身到 3 头身。
- 武器、发型、披风、法器夸张化。
- 动作幅度大，移动/攻击/受击反馈明显。
- 国风三国元素 + 星象纹样。

禁止：

- 幼儿化大头萌宠。
- 低龄糖果色、现代卡通玩具感。
- 武将全部变成无威严表情包。
- 背景过暗、竞技场化、压迫感过强。
- 战斗背景中央大面积纯白高光或厚云，导致棋盘、单位数字、顶部状态和底部手牌不可读。
- 把战斗背景画成完整舞台、祭坛、平台、圆形竞技场或带观众席的场景；棋盘和交互层必须由 Godot 生成。

## 2. 统一画面基准

| 项目 | 规格 |
| --- | --- |
| 设计基准 | 横屏 `2400 × 1080` |
| 小屏兼容 | 横屏 `1600 × 720` |
| 战斗视角 | 2D 横屏，棋盘略俯视，Q版小人站在格子内 |

| 背景职责 | 提供星空氛围，不承担战斗爽感，不抢棋盘/角色/特效 |

| 接入路径 | `assets/art/characters/`、`assets/art/effects/`、`assets/art/board/`、`assets/art/ui/`、`assets/art/backgrounds/` |

## 4. 动作与反馈规格

### 4.1 移动

- Q版武将从格子中心滑动到目标格。
- 移动时可加轻微上下弹跳和星尘拖尾。
- 快速单位/突进技能可使用更长的青白流光轨迹。
- 移动落点要清楚，不允许角色拖尾盖住目标格判断。

### 4.2 普攻

- 近战：角色向目标方向前冲 8–18 px，播放斩击弧线，命中闪光，目标抖动。
- 远程/法术：角色抬手/挥扇，发射小弹体或星火，命中后爆点。
- 攻击结束回到格子中心，不能造成站位误读。

### 4.3 技能

- 技能触发必须比普攻高一级：角色短暂放大、特效颜色更强、伤害/状态飘字更醒目。
- 每类技能有统一视觉语言：
  - 突进：青白流光 + 残影。
  - 火焰：赤金爆燃 + 灼烧状态小图标。
  - 雷法：黄紫雷纹 + 从上落下。
  - 护盾：金色圆盾/阵纹展开。
  - 召唤：星门/阵盘生成。

### 4.4 受击与击杀

- 受击：目标闪白 1 次 + 小幅抖动 + 伤害数字跳出。
- 格挡/减伤：显示 `格挡` 或盾形闪光，不用长文字。
- 击杀：目标缩小/后仰/化星屑淡出，格子恢复可见。
- 奕星师受伤：左右奕星师面板闪烁，HP 数字跳动。

## 6. 图像生成提示词约束

正式产图时统一遵守：

- 题材：三国名将、星图棋局、奕星师、天命、星轨、国风幻想。
- 画风：英气 Q 版、2.5–3 头身、清晰轮廓、动作夸张、移动端可读。
- 战斗角色：必须适合在 10×5 棋盘上移动、攻击、受击和释放技能。
- 背景：开阔、明亮、低干扰，不抢角色和特效。
- 禁止：半写实成人比例作为默认战斗单位、低龄萌宠化、现代机甲、厚重暗黑竞技场、把文字/数字画进图片。
- UI 切图：透明背景、干净边缘、可九宫格拉伸区域明确。

示例方向词：

```text
英气Q版三国武将小人，2.5头身，夸张武器，清晰轮廓，国风星象纹样，适合手机战棋格子上移动攻击，transparent background, no text
```

## 7. 导出与接入要求

1. 每个角色交付：透明 PNG 静帧或序列 + 源文件/提示词记录。
2. 面向右为标准方向，Godot 可镜像面向左；特殊武器如弓/长枪需检查镜像后是否合理。
3. 角色不要带烘焙血条、名字、伤害数字；这些由 Godot UI 层生成。
4. 技能特效必须透明背景，中心点/起点明确，便于对齐棋盘格。
5. 背景图中央棋盘区域保持低对比，不与 Q版小人、伤害数字、技能特效竞争。
6. 所有新增资源必须避免放入 `builds/`；截图和临时产物继续放系统临时目录。

---

## 8. 建议执行顺序

1. 先让图像专家确认 `A00 Q版星图战棋风格参考图`。
2. 再做 1 名武将 Q版小人样张：建议关羽或赵云，验证棋盘站位、攻击姿态和可读性。
3. 同步做 3 类通用特效样张：斩击、火焰、护盾。
4. 再做首批 6 名 Q版武将待机图。
5. 最后做 UI 皮肤和背景统一，避免背景先行导致战斗反馈不足。

---

## 9. 当前非目标

- 暂不一次性量产 42 名武将。
- 暂不要求完整 Spine 骨骼动画。
- 暂不做写实/半写实全身战斗单位。
- 暂不把精确 10×5 棋盘交给图片模型生成；棋盘仍由 Godot 稳定叠加。
