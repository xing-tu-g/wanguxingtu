# 万古星图：数值与内容表

版本：v0.1-MVP  
用途：集中存放职业、地形、策略卡、武将、经济等可数据化内容，供 Godot 数据表或资源文件实现。

---

## 1. 数值状态说明

当前所有技能与数值均为原型验证数值，不代表最终上线数值。

实现建议：

- 用数据表驱动，不把数值硬编码在战斗逻辑里。
- 所有数值字段保留 `design_note` 或 `is_placeholder` 标记。
- 先保证规则可跑通，再进行平衡。

## 2. 职业标准模型

| 职业 | 攻击距离 | 攻击力类型 | 血量 | 移动 | 物理格挡 | 法术格挡 | 伤害类型 | 定位 |
|---|---:|---|---:|---:|---:|---:|---|---|
| 射手 | 3–6 | 物理攻击力 3–7 | 2–5 | 1–3 | 0 | 0 | 物理 | 远程、高伤、破甲、群伤 |
| 坦克 | 1 | 物理攻击力 1–3 | 4–10 | 1–3 | 1–3 | 0 | 物理 | 身板硬、保护队友 |
| 武卫 | 2 | 物理攻击力 2–4 | 3–7 | 2–3 | 0 | 0 | 物理 | 稳步推进、克制战士 |
| 战士 | 1 | 物理攻击力 3–6 | 4–8 | 3–5 | 0 | 0 | 物理 | 快速冲锋、近战爆发 |
| 法师 | 3–5 | 法术攻击力 3–6 | 4–8 | 1–3 | 0 | 0 | 法术 | 法术伤害、控制、治疗 |
| 刺客 | 1 | 物理攻击力 1–4 | 4–8 | 3–6 | 0 | 0 | 物理 | 无视阻挡、后排干扰 |

设计约束：

- 属性越高，召唤费用越高。
- 技能越强，基础属性应适度压低。
- 同一传说武将通常取职业区间中上值。
- 射手和法师脆弱，但能通过射程和地形获得优势。

## 7. 地形表

| 地形 | 是否可通行 | 是否可部署 | 效果 |
|---|---|---|---|
| 草地 | 是 | 仅己方部署区 | 无效果 |
| 泥沼 | 是 | 仅己方部署区 | 非刺客 / 战士移动时多消耗 1 移动力 |
| 河流 | 是 | 仅己方部署区 | 站在河流上攻击力 -1，被攻击时受到伤害 +1，攻击奕星师伤害 -1 |
| 高地 | 是 | 仅己方部署区 | 射手 / 法师射程 +1 |

随机规则：

- 每局生成 5 个特殊地形。
- 双方部署区各 1 个。
- 公共区域 3 个。
- 不覆盖已部署单位；开局前生成。

## 9. 对战奖励修订

MVP 阶段先做奖励展示占位，可不接完整经济。

修订原则：

- 奕星师经验不由战斗胜负直接获得。
- 对战奖励主要给铜钱、券类、天梯分。
- 抽卡系统暂缓，因此券类可先记录但不开放消耗。

| 结果 | 铜钱 | 奕星师经验 | 券类 | 天梯 |
|---|---:|---:|---|---|
| 胜利 | 180 | 0 | 稀有券概率占位 | 增加 |
| 失败 | 80 | 0 | 稀有券概率占位 | 减少 |
| 每日首胜 | +1000 | 0 | 传说券占位 | — |
| 连胜 3+ | +90 | 0 | 稀有券占位 | — |

## 10. 本场战斗统计字段

建议 Godot 侧每个战斗单位实例维护以下统计：

| 字段 | 含义 |
|---|---|
| `kills` | 击杀敌方单位数量 |
| `damage_to_units` | 对敌方单位造成的总伤害 |
| `damage_to_master` | 对敌方奕星师造成的总伤害 |
| `damage_taken` | 承受伤害总量 |
| `healing_done` | 治疗量 |
| `summons_created` | 召唤单位数量 |
| `control_applied` | 控制次数 |
| `turns_alive` | 存活回合数 |
| `tiles_moved` | 移动格数 |

当前只展示统计，不评选 MVP。
