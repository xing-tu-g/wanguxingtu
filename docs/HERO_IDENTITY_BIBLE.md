# Hero Identity Sprint v1

日期：2026-07-04

目标：把 55 名可玩武将从“数值单位”升级为“一句话能记住的战斗角色”。本文件只定义 Battle MVP 内的英雄身份，不新增职业、系统、资源、UI、美术、AI 或经济系统。

## 设计边界

- 只使用现有触发：`deploy`、`turn_start`、`attack_hit`、`passive`。
- 只使用现有效果：伤害、范围伤害、治疗、护盾、眩晕、缓速、加攻、成长、召唤、移动、全军压制、相邻援护。
- 所有数值保持固定值，不使用百分比、倍率、概率、暴击率。
- 英雄身份优先于微小数值差异；如果遮住名字仍能猜出人物，说明方向合格。

## 模拟验证摘要

本阶段使用 `scripts/tools/run_hero_identity_simulation_v1.gd` 重新模拟 100 局，统计：

- 技能触发率：`skill_triggers / appearances`。
- 技能存在感：`skill_triggers + damage + healing + tanking * 0.5`。
- 低存在感英雄：用于后续观察，不直接等价于弱。
- 高存在感英雄：用于观察是否有身份过强或过亮。
- 完整技能签名重复：本轮目标为 0 组。

当前需要重点观察的低存在感英雄通常是低触发近战或纯防守单位，例如赵云、华雄、马超、徐盛、张飞。这不一定是平衡问题，但后续实机体验里要确认玩家是否能感知到其身份。

## 需要重新设计

本轮已经完成第一轮重新设计，当前没有必须立即重做的英雄。

后续如果出现以下情况，再进入 Hero Identity Sprint v2：

- 模拟中长期低触发且实机无感。
- 玩家只看技能仍无法猜到人物。
- 某职业再次退化成同一种模板。
- 某阵营特色被单个英雄过度代表，导致其它同阵营英雄无存在感。
