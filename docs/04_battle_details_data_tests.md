# 万古星图：战斗细则、数据结构与测试清单

版本：v0.1-MVP  
用途：补充核心规则中的实现细节，降低 Hermes + Codex + Godot 开发歧义。

---

## 1. 单位行动顺序

### 1.1 行动单位范围

当前行动方回合内，仅当前行动方场上的单位行动。

单位依次行动，不同时行动。

### 1.2 行动顺序

多个单位按“前方单位优先，后方单位随后”的规则行动。

以左侧玩家向右进攻为例：

- 列号大的单位更靠前，优先行动。
- 列号小的单位更靠后，后行动。

以右侧玩家向左进攻为例：

- 列号小的单位更靠前，优先行动。
- 列号大的单位更靠后，后行动。

如果多个单位处在同一前后位置，则按行号从小到大行动；如果仍需稳定排序，则按单位入场顺序行动。

程序排序建议：

```text
左侧玩家：列号降序 → 行号升序 → 入场序号升序
右侧玩家：列号升序 → 行号升序 → 入场序号升序
```

---

## 2. 单位行动流程

每个单位行动时按以下顺序执行：

1. 判定攻击范围内是否存在敌方单位。
2. 如果存在敌方单位：直接攻击，不移动。
3. 如果不存在敌方单位：尝试向前移动。
4. 移动结束后，再次判定攻击范围。
5. 如果移动后存在敌方单位：攻击敌方单位。
6. 如果移动后没有敌方单位，但敌方奕星师在攻击范围内，且前方无敌方单位：攻击敌方奕星师。
7. 如果仍无可攻击目标：行动结束。

### 2.1 攻击先于移动

单位每次行动时必须先判定攻击。

只要攻击范围内有敌方单位，就直接攻击，不会为了靠近奕星师或更优位置而移动。

### 2.2 移动后攻击

如果攻击前没有可攻击敌方单位，单位会先向前移动，再进行一次攻击判定。

移动后攻击同样优先攻击敌方单位；只有没有敌方单位且满足奕星师攻击条件时，才攻击奕星师。

## 3. 移动细则

### 3.1 移动方向

单位只允许向前移动。

- 左侧玩家单位：只能向列号增加方向移动。
- 右侧玩家单位：只能向列号减少方向移动。
- 不允许斜向移动。
- 不允许横向移动。
- 不允许后退。

### 3.2 格子占用

每个格子仅能存在一个单位。

单位包括：

- 玩家放置武将。
- 敌方放置武将。
- 召唤物。
- 残影或其他可被攻击实体。

### 3.3 阻挡规则

除刺客或拥有穿越技能的单位外，其他单位遇到阻挡单位时停止移动。

阻挡单位包括：

- 敌方单位。
- 友方单位。
- 召唤物。
- 残影或其他占格实体。

MVP 阶段不做绕路。

### 3.4 刺客穿越

刺客可以穿过阻挡单位继续向前移动，但仍不能停留在已占用格子上。

如果刺客移动结束位置被占用，则停在该位置之前最近的空格。

如果刺客没有可停留空格，则保持原地。

## 4. 攻击距离与目标选择

### 4.1 距离算法

所有攻击距离统一使用曼哈顿距离。

```text
距离 = abs(攻击者列 - 目标列) + abs(攻击者行 - 目标行)
```

当距离 <= 攻击者射程时，目标处于攻击范围内。

### 4.2 敌方单位目标选择

如果攻击范围内存在多个敌方单位，优先攻击距离我方阵地最近的敌方单位。

程序定义：

- 左侧玩家：优先攻击列号更小的敌方单位。
- 右侧玩家：优先攻击列号更大的敌方单位。
- 若列号相同，优先攻击 HP 更低的单位。
- 若仍并列，按行号升序。
- 若仍并列，按入场序号升序。

### 4.3 奕星师攻击条件

单位可以攻击敌方奕星师，需同时满足：

1. 攻击范围内没有敌方单位。
2. 单位前方没有敌方单位阻挡。
3. 敌方奕星师在攻击范围之内。

奕星师位置建议：

- 左侧玩家奕星师位于棋盘左侧外一列，可抽象为列 `0`。
- 右侧玩家奕星师位于棋盘右侧外一列，可抽象为列 `11`。
- 行坐标可按单位当前行计算，或抽象为覆盖全行目标。

MVP 推荐实现：

- 奕星师视为位于敌方底线外侧，覆盖所有行。
- 对左侧玩家单位，若 `11 - unit_col <= range`，则敌方奕星师在范围内。
- 对右侧玩家单位，若 `unit_col - 0 <= range`，则敌方奕星师在范围内。

## 5. 技能触发与效果规则

### 5.1 技能触发原则

技能不需要额外优先级系统；在对应事件发生时立即触发。

常见事件：

| 触发事件 | 说明 |
|---|---|
| 部署时 | 单位被放到棋盘后立即触发 |
| 回合开始 | 当前行动方回合开始时触发 |
| 行动前 | 单位准备执行行动前触发 |
| 攻击前 | 单位确认攻击目标后、造成伤害前触发 |
| 攻击命中 | 单位造成伤害后触发 |
| 击杀后 | 本次攻击导致目标死亡后触发 |
| 受击后 | 单位承受伤害后触发 |
| 死亡时 | 单位 HP <= 0 时触发 |
| 回合结束 | 当前行动方回合结束时触发 |

### 5.2 同一效果不可重复

同一单位身上的同一效果不可重复叠加。

示例：

- 同一单位已经有“攻击力 +2”，再次获得同名“攻击力 +2”时，不叠加数值。
- 同一单位已经有“沉默”，再次获得沉默时，不延长持续时间。
- 不同来源但同名效果按同一效果处理。

### 5.3 效果持续时间

MVP 阶段所有临时效果均只持续一回合。

一回合定义：

- 如果效果在当前行动方回合内获得，则在当前行动方回合结束时移除。
- 如果效果明确作用于敌方下一次行动，则在敌方对应单位完成一次行动后移除。

为简化实现，MVP 默认使用“当前回合结束移除”。

### 5.4 状态实现建议

状态以列表形式挂在单位实例上：

```json
{
  "id": "burn",
  "name": "燃烧",
  "duration_turns": 1,
  "source_unit_id": "unit_001",
  "value": 3
}
```

## 6. 召唤物规则

召唤物与放置单位规则一致，无特殊性。

召唤物：

- 占用棋盘格子。
- 可移动。
- 可攻击。
- 可被攻击。
- 可被治疗。
- 可被控制。
- 可触发击杀统计。
- 死亡时按普通单位处理。

如果召唤位置被占用，则该召唤物不生成。

---

## 7. AI 对手

MVP 阶段可以先不做智能 AI。

推荐占位方案：

- 使用预设敌方阵容。
- 敌方回合自动选择当前可支付费用中费用最高的单位。
- 部署位置选择敌方部署区最靠前的空格。
- 如果无法部署，则跳过部署阶段。

如果实现成本仍高，可以先做“自动布敌方”按钮，由玩家手动触发敌方简化部署。

## 8. 战斗日志格式

战斗日志统一格式：

```text
第 N 回合｜单位｜行为｜结果
```

示例：

```text
第 1 回合｜关羽｜部署｜消耗 5 星力，进入 (2,3)
第 1 回合｜关羽｜移动｜从 (2,3) 前进到 (5,3)
第 1 回合｜关羽｜攻击｜对张辽造成 4 点物理伤害
第 2 回合｜周瑜｜攻击命中｜张辽获得燃烧 1 回合
第 3 回合｜张角｜召唤｜生成黄巾兵 ×3
第 5 回合｜孙尚香｜攻击奕星师｜造成 3 点伤害
```

日志字段建议：

| 字段 | 示例 |
|---|---|
| `turn` | 1 |
| `actor` | 关羽 |
| `action` | 攻击 |
| `result` | 对张辽造成 4 点物理伤害 |
| `tags` | `["damage", "physical"]` |

## 9. 阵容、手牌与 draw_queue

### 9.1 阵容数量

战前阵容最多 15 张卡牌。

### 9.2 同名卡数量限制

| 品质 | 同名武将上阵上限 |
|---|---:|
| 传说 | 1 |
| 史诗 | 2 |
| 精英 | 3 |

### 9.3 开局洗牌与初始手牌

开局从玩家卡组随机洗牌，生成 `draw_queue`。

初始从 `draw_queue` 抽取 5 张卡牌，填充 `hand_slots`。

### 9.4 上场后立即补位

玩家上场 1 张卡牌后，该手牌槽位立即从 `draw_queue` 抽取下一张卡牌补位。

手牌始终尽量保持 5 张，但只由部署后的自动补位维持，不存在手动换牌。

### 9.5 卡组抽完

当 `draw_queue` 为空后，被使用的槽位不再补牌。

空槽可显示为空槽或“牌库已空”，弃牌不回收进 `draw_queue`。

### 9.6 禁用操作

普通战斗流程中不存在以下操作：

- 无刷新。
- 无锁定。
- 无重置。
- 无手动换牌。

## 10. 胜负边界

### 10.1 正常胜负

任一方奕星师 HP <= 0 时，战斗结束。

### 10.2 双方同时失败

如果同一次结算后双方奕星师 HP 均 <= 0，判定双方均失败。

结算页显示：

```text
双方奕星师同时归零，本局双方失败。
```

### 10.3 暂不处理项

MVP 暂不考虑：

- 回合过长强制结束。
- 平局保护。
- 投降。
- 断线重连。
