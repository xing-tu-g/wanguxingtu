# 万古星图：战斗界面对弈式布局设计稿

版本：v0.1-M37-layout  
目标：把战斗画面从“调试信息面板”升级为“平面星图棋盘 + 左右奕星师对弈 + 英气 Q 版武将小人战斗表现”的可执行 UI / 美术布局规范。

---

## 1. 核心画面设想

战斗界面应优先呈现一个明确的对弈场景：

- 中央是一张横向平面星图棋盘，10 列 × 5 行。
- 武将单位站在棋盘格上，而不是漂浮在 UI 表格里。
- 左侧站我方奕星师，右侧站敌方奕星师，二者像棋手一样隔着棋盘对局。
- 玩家视角是俯视略带正交透视的 2D 横屏画面：棋盘平铺，武将以 2.5–3 头身英气 Q 版小人站在格子内，能移动、攻击、受击和释放技能。
- UI 信息服务于“看懂对局”，不能抢走棋盘主体面积。

推荐关键词：英气 Q 版、星图战棋、三国武将小人、夸张武器、技能特效、星盘、棋局、奕星师、星轨、天命、琉璃面板、开阔星空背景。

---

## 2. 横屏尺寸基准

当前 AVD / 高端安卓横屏验证尺寸：`2400 × 1080`。  
同时必须兼容较小横屏：`1600 × 720`。

### 2.1 安全区域

| 分辨率 | 安全边距 | 可用区域 |
| --- | --- | --- |
| 2400×1080 | 左右 48，上下 27 | 2304×1026 |
| 2340×1080 | 左右 47，上下 27 | 2246×1026 |
| 1920×1080 | 左右 38，上下 27 | 1844×1026 |
| 1600×720 | 左右 32，上下 18 | 1536×684 |

设计时以比例为准，不写死绝对像素；Godot 实现用 `Control` 容器和最小尺寸约束。

---

## 3. 推荐总体构图

采用“三层画面 + 两侧角色”的布局：

```text
┌────────────────────────────────────────────────────────────┐
│ 顶部状态栏：回合 / 星潮 / 双方 HP 星力 / 设置返回             │
├──────────┬──────────────────────────────────────┬──────────┤
│ 我方奕星师 │              星图棋盘 10×5              │ 敌方奕星师 │
│ Q版操盘者 │     Q版武将小人站格、移动、攻击、放技能    │ Q版操盘者 │
│ 简短状态  │                                      │ 简短状态  │
├──────────┴──────────────────────────────────────┴──────────┤
│ 底部手牌条：我方手牌卡牌 / 当前选中 / 推进回合按钮             │
└────────────────────────────────────────────────────────────┘
```

### 3.1 2400×1080 建议比例

| 区域 | 位置 | 建议尺寸 | 用途 |
| --- | --- | --- | --- |
| 顶部状态栏 | 顶部 | 高 120–150 | 回合、星潮、双方资源摘要 |
| 左奕星师区 | 左侧 | 宽 260–320 | 我方奕星师立绘、HP、星力、状态 |
| 右奕星师区 | 右侧 | 宽 260–320 | 敌方奕星师立绘、HP、星力、状态 |
| 中央棋盘区 | 中央 | 约 1680×650 | 10×5 平面棋盘、武将单位 |
| 底部手牌区 | 底部 | 高 150–180 | 手牌、费用、部署按钮、推进回合 |
| 浮层/抽屉 | 覆盖层 | 不参与主布局 | 单位详情、牌库弃牌、日志、规则 |

### 3.2 棋盘格尺寸建议

在 `2400×1080` 下，棋盘可用约 `1680×650`：

- 单格宽：约 `160–168`。
- 单格高：约 `120–130`。
- 武将站位：Q 版小人脚底中心落在格子中心偏下，身体不能遮挡相邻格判断。
- 部署区：左 1–3 列用蓝色星轨边框；中 4–7 列用中性星云色；右 8–10 列用红/紫色星轨边框。

在 `1600×720` 下，棋盘可用约 `1120×420`：

- 单格宽：约 `108–112`。
- 单格高：约 `78–84`。
- 小屏应降低单位文字密度，优先显示图标/血条，不显示长文本。

## 4. 常驻显示内容

常驻内容必须一眼可读，且不能遮挡棋盘。

### 4.1 顶部状态栏

常驻显示：

- 当前回合：`第 N 回合`。
- 当前行动方：`我方行动` / `敌方行动`。
- 星潮阶段：未开启时简写；开启后显示 `星潮 I/II/III`。
- 简短提示：如 `选择手牌 → 点击左侧部署区`。
- 返回首页 / 设置 / 暂停按钮。

不常驻显示：

- 完整日志。
- 长规则说明。
- 完整统计。

### 4.2 左右奕星师区

每侧常驻显示：

- 奕星师立绘或半身剪影。
- HP 条：大号血条 + 数字。
- 星力：星点或数字。
- 牌库 / 手牌 / 弃牌数量：只显示数字，不展开牌名。
- 状态图标：燃烧、增益、星潮影响等。

表现建议：

- 我方奕星师在左侧朝右，敌方奕星师在右侧朝左。
- 受到奕星师伤害时，角色区闪烁或血条抖动。
- 当前行动方的奕星师区加金色外发光。

### 4.3 中央棋盘区

常驻显示：

- 10×5 棋盘。
- Q 版武将小人。
- 单位生命数字 / 状态图标；正式表现可配短血条，但不能遮挡动作。
- 当前可部署格高亮。
- 最近行动格高亮。
- 当前选中单位边框。
- 移动、普攻、技能、受击、击杀的短反馈。

不建议常驻显示：

- 每个单位的完整属性文字。
- 每格坐标文字，正式 UI 应隐藏；调试模式可开关。
- 每个技能的长描述。

### 4.4 底部手牌区

常驻显示：

- 我方手牌卡牌横向排列。
- 每张卡：费用、武将名、阵营标识、Q 版武将小像。
- 当前选中卡牌放大或上浮。
- `推进回合` 主按钮。
- 普通战斗 UI 不显示重置；调试入口不得出现在玩家界面。

不常驻显示：

- 敌方完整手牌牌名。
- 我方完整弃牌列表。
- 完整牌面说明。

## 5. 子窗口 / 抽屉设计

以下内容应作为子窗口、浮层或抽屉，不参与主战斗布局高度分配。

### 5.1 单位详情浮层

触发：点击棋盘上的单位。

显示：

- 武将名、阵营、关键战斗属性。
- HP、攻击、射程、移动、物理/法术格挡。
- 技能名与短描述。
- 当前状态。

布局：

- 靠近被点击单位弹出，但不能遮挡太多棋盘。
- 小屏可固定右上或右侧抽屉。

### 5.2 牌区抽屉

触发：点击底部或顶部 `牌区` 按钮。

显示：

- 我方手牌、我方弃牌、敌方公开牌区。
- 牌库剩余数量与弃牌回收提示。
- 点击卡牌后显示牌面说明。

规则：

- 抽屉覆盖在棋盘上方或从底部滑出。
- 不挤压棋盘。
- 必须有关闭按钮或点击遮罩关闭。
- 当前 M36 已完成基础覆盖式抽屉，后续应补关闭按钮和更像卡牌的横向布局。

### 5.3 战斗日志抽屉

触发：点击 `日志`。

显示：

- 最近行动列表。
- 技能触发。
- 抽牌 / 洗牌 / 部署记录。
- 可筛选：全部、伤害、技能、抽牌。

常驻替代：主界面只显示最近 1–2 条事件。

### 5.4 规则 / 地形说明浮层

触发：点击地形格、状态图标或问号。

显示：

- 地形效果。
- 状态效果。
- 星潮规则。

## 6. 信息取舍规则

### 必须第一优先级

1. 棋盘是否清楚。
2. 武将站在哪个格子。
3. 当前谁行动。
4. 玩家能部署哪些手牌。
5. 当前选中什么。
6. 是否能推进回合。

### 第二优先级

1. 双方 HP / 星力。
2. 牌库 / 手牌 / 弃牌数量。
3. 最近发生了什么。
4. 单位血量。
5. 技能触发提示。

### 第三优先级

1. 完整日志。
2. 完整属性。
3. 完整牌区。
4. 战斗统计。
5. 规则说明。

第三优先级全部进入子窗口。

## 7. Q版美术与战斗表现说明

如果要让美术先出图，建议优先出“Q版战斗表现参考图”，而不是只出漂亮背景。背景负责氛围，Q版武将动作和技能特效负责爽感。

### 7.1 需要美术提供

1. 横屏 Q版战斗表现参考图：`2400×1080`。
2. 中央平面星图棋盘：10×5 格，左右部署区明显。
3. 左右 Q版奕星师站位：左侧朝右，右侧朝左。
4. Q版武将小人站在格上的视觉示例：至少 2 个我方、2 个敌方。
5. 普攻/技能/受击反馈示例：斩击、火焰、护盾或雷法至少三选一。
6. 底部手牌条样式：5 张手牌 + 自动补牌/空槽示意，使用 Q版武将小像。
7. 顶部状态栏样式。
8. 子窗口样式：单位详情 / 牌区抽屉 / 日志抽屉三选一即可。

### 7.2 美术风格描述

- 整体：英气 Q 版星图战棋，不幼稚、不低龄，适合武将移动攻击和释放技能。
- 背景：开阔蓝紫星空、淡淡星云、局部金色星轨；背景必须低干扰。
- 棋盘：像悬浮星盘或玉石棋盘，格线是发光星轨，不要太厚。
- 部署区：我方蓝金，敌方紫红，中间公共区偏青绿或暗金。
- 奕星师：Q版/半Q版操盘者，不是普通武将；衣袍带星纹，手持星盘或棋子。
- 武将：2.5–3 头身英气 Q 版小人，武器和轮廓夸张，能承载移动、普攻、技能、受击、退场。
- 特效：斩击、突进、火焰、雷法、护盾、召唤要有统一颜色语言，不能只靠文字提示。
- UI 面板：半透明深色琉璃底，金色细边，避免大块纯黑。

### 7.3 美术不要先做死的内容

- 不要先把所有文字画死在图里。
- 不要把手牌、日志、详情都画成常驻大面板。
- 不要让棋盘小于画面宽度的 60%。
- 不要让奕星师遮挡部署区前三列。
- 不要把武将做成静态头像棋子作为最终方向；头像可作临时占位，但正式战斗需要 Q版小人动作。
- 不要让背景或大立绘抢走技能特效和单位动作的可读性。

## 8. Godot 实现建议

建议 M37 做灰盒布局，而不是直接换美术。

### M37：对弈式战斗灰盒布局

目标：把当前调试式 `BattleScreen` 改成“左右奕星师 + 中央棋盘 + 底部手牌”的结构。

建议场景结构：

```text
BattleScreen
  Background
  Margin
    MainLayout
      TopStatusBar
      DuelArea
        LeftMasterPanel
        BoardPanel
        RightMasterPanel
      BottomHandBar
  UnitDetailPopup
  CardZoneDrawer
  BattleLogDrawer
```

验收：

- 2400×1080 下棋盘和左右奕星师同时可见。
- 1600×720 下棋盘仍可操作，手牌条不遮挡棋盘。
- 展开牌区 / 单位详情 / 日志不改变主布局尺寸。
- 旧 M1–M36 规则测试继续通过。

### 后续：Q版战斗表现主题化

目标：把灰盒替换为 Q版星图战棋表现资源，优先接入武将小人、动作反馈和技能特效，再统一 UI/背景。

验收：

- 不改变布局结构。
- Q版武将移动/攻击/受击时仍能看清所在格。
- 技能特效有爽感但不遮挡关键 UI。
- 所有按钮可点区域保持足够大。
- AVD 截图确认棋盘、奕星师、手牌条、战斗反馈层级清晰。

---

## 9. 当前决策

- 战斗主视觉采用“平面棋盘 + 左右奕星师对弈 + Q版武将小人战斗”。
- 棋盘是主角，UI 面板只能辅助。
- 武将动作、技能特效、命中/受击反馈是爽感核心，背景只做低干扰舞台。
- 牌区、日志、单位详情、规则说明都做子窗口/抽屉。
- 下一步优先让图像专家优化 Q版战斗表现基准图，再回到 Godot 做资源接入与动效原型。
