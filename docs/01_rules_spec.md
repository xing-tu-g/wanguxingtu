## 1. 产品核心

### 1.1 核心乐趣假设

在华夏文明的星图上，用三国名将的魂棋进行一场自动战棋对决：选将、布阵、看他们在随机地形上自动厮杀。赢下一局，代表奕星师将一颗陨落的将星重新拉回文明星图。

### 1.2 设计支柱

1. 星图设定：所有玩法围绕“星图”“魂星”“天缺”展开，不做换皮三国。
2. 易学不易精：8–15 分钟上手，但阵容搭配和地形利用需要长期钻研。
3. 每局新棋局：随机地形和随机策略卡让每局产生变化。
4. 降肝降氪：每日首胜和前几局收益更高，后续收益递减。
5. 真实概率：抽卡概率透明公示，保底机制清晰；MVP 阶段先占位，不开发完整抽卡。

### 1.3 MVP 产品定位

- 类型：休闲 + 轻策略 + 自动战棋。
- 单局目标：8–15 分钟。
- 初期模式：本地回合制 PVE 或本地模拟对战。
- 暂缓内容：联网 PVP、真实充值、完整抽卡、完整排位赛季。

## 2. 术语统一

### 2.1 奕星师

原文中所有“弈天师”统一改为“奕星师”。

奕星师是双方玩家在战斗中的核心血量目标：

- 双方各一名奕星师。
- 奕星师不占棋盘格子。
- 奕星师不能移动、不能攻击、没有固有减伤。
- 先将敌方奕星师 HP 降至 0 的一方获胜。

### 2.2 战斗初始 HP

战斗初始 HP = 当前奕星师等级对应的生命值。

示例：

| 奕星师等级 | 战斗初始 HP |
|---|---:|
| 1 | 30 |
| 2 | 36 |
| 3 | 42 |
| 4 | 48 |
| 5 | 54 |
| 6 | 60 |
| 7 | 66 |
| 8 | 72 |
| 9 | 78 |
| 10 | 84 |

## 3. 棋盘与区域

### 3.1 棋盘尺寸

- 棋盘为 10 列 × 5 行。
- 推荐坐标：列 `1–10`，行 `1–5`。
- 也可在程序内部使用 0 基坐标，但 UI 和策划文档统一展示 1 基坐标。

### 3.2 区域定义

以玩家 A 在左、玩家 B 在右为默认视角：

| 区域 | 玩家 A 视角列 | 玩家 B 视角列 | 用途 |
|---|---:|---:|---|
| A 方部署区 | 1–3 | 8–10 | A 方可部署武将 |
| 公共区域 | 4–7 | 4–7 | 双方单位移动、交战区域，不允许直接部署 |
| B 方部署区 | 8–10 | 1–3 | B 方可部署武将 |

描述建议：

> 棋盘共 10 列 × 5 行。双方各拥有靠近己方奕星师的前三列作为部署区；中间四列为公共区域。玩家只能在己方部署区的空格部署武将，单位进入战斗后可移动到公共区域和敌方部署区。

### 3.3 部署规则

- 玩家只能在己方部署区内的空格部署武将。
- 部署消耗星力，消耗值等于武将召唤费用。
- 公共区域和敌方部署区不能直接部署。
- 已被单位、召唤物、残影等占据的格子不能部署。
- 如果部署区无空格，则本回合不能部署新武将。

## 4. 回合制流程

### 4.1 对战性质

本游戏战斗为回合制自动战棋。

- 双方按先后手轮流行动。
- 每个玩家回合内，玩家只负责放置武将或使用策略卡。
- 武将移动和攻击由系统自动执行。
- 当前 MVP 不做即时操作、不做同步实时 PVP。

### 4.2 开局流程

1. 抽取先后手。
2. 先手初始 5 星力。
3. 后手初始 6 星力。
4. 根据双方初始卡组 / 候选武将进入第 1 回合。

### 4.3 单方回合流程

每个玩家回合按以下顺序执行：

1. 回合开始：恢复星力，触发回合开始类效果。
2. 放置阶段：玩家可放置武将，消耗星力。
3. 策略阶段：如果有可用策略卡，可选择使用。
4. 自动移动：己方场上武将按行动规则自动移动。
5. 自动攻击：己方场上武将按攻击规则自动攻击。
6. 回合结束：结算燃烧、持续状态、死亡、召唤等效果。
7. 检查胜负：任一奕星师 HP <= 0 时结束战斗。

### 4.4 循环结构

```text
先后手抽取
└─ 先手初始 5 星力，后手初始 6 星力

先手方回合：
├─ 放置武将（消耗星力）
├─ 武将自动移动
├─ 武将自动攻击
└─ 回合结束

后手方回合：
├─ 放置武将（消耗星力）
├─ 武将自动移动
├─ 武将自动攻击
└─ 回合结束

循环，直到一方奕星师 HP = 0
```

## 5. 星力与星潮

### 5.1 星力

- 星力是奕星师召唤武将与发动战术的战场能量。
- 星力上限：10。
- 先手初始星力：5。
- 后手初始星力：6。
- 基础每回合恢复 2 点星力。

### 5.2 星潮

- 每 3 个完整回合进入一次星潮阶段。
- 星潮后，后续每回合星力恢复量 +1。
- 星潮时额外抽取 1 张策略卡。
- 从第 8 / 12 / 16 ... 回合起，单位攻击奕星师时伤害分别 +1 / +2 / +3 ...。
- 星潮伤害加成只影响对奕星师的伤害，不影响单位对单位的战斗。

## 6. 移动规则

### 6.1 基础移动

- 单位每次行动时，最多移动等于自身移动值的格子数。
- 默认前进方向：朝敌方奕星师方向前进。
- 单位只允许向前移动，不允许斜向、横向或后退。
- 单位行动时先判定攻击；若攻击范围内没有敌方单位，才向前移动。
- 单位不能移动到已被占据的格子。

### 6.2 阻挡

- 普通单位不能穿过敌方单位。
- 普通单位不能穿过友方单位。
- 除刺客或具有“穿越 / 无视阻挡”技能的单位外，单位遇到阻挡单位时停止移动。
- MVP 阶段不做绕路。
- 如果没有合法前进位置，单位停留原地。

### 6.3 棋盘满格处理

MVP 规则：

- 召唤物生成时，如果目标格无空位，则该召唤物不生成。
- 部署区满时，玩家不能继续部署。
- 死亡残影类效果如果原地无法保留，则不生成残影。

---

## 7. 攻击规则

### 7.1 攻击范围

- 所有攻击距离统一使用曼哈顿距离。
- 当 `abs(攻击者列 - 目标列) + abs(攻击者行 - 目标行) <= 攻击者射程` 时，目标处于攻击范围内。
- 坦克、战士、武卫、刺客默认攻击单体目标。
- 攻击距离由武将面板定义。

### 7.2 目标优先级

当攻击范围内存在多个敌方单位时，优先攻击“距离我方阵地最近”的敌方单位。

建议程序化定义：

- 对左侧玩家而言，优先攻击列号更小的敌方单位。
- 对右侧玩家而言，优先攻击列号更大的敌方单位。
- 如果仍有多个目标并列，则优先攻击 HP 最低者。
- 如果仍并列，则按固定顺序选择，确保可复现。

### 7.3 攻击奕星师

- 当攻击范围内没有敌方单位，且单位已推进到可以攻击敌方奕星师的位置时，单位攻击敌方奕星师。
- 奕星师没有物理 / 法术格挡。
- 单位对奕星师造成全额伤害，但受河流、星潮等规则修正。

### 7.4 伤害类型

- 物理伤害：受物理格挡减免。
- 法术伤害：受法术格挡减免。
- 真实伤害：无视两类格挡。
- 伤害最低为 0。

## 8. 地形规则

每局随机生成 5 个特殊地形：

- 双方半场各 1 个。
- 公共区域 3 个。
- 其余为草地。

| 地形 | 效果 |
|---|---|
| 草地 | 无效果 |
| 泥沼 | 非刺客 / 战士移动时多消耗 1 移动力 |
| 河流 | 可通行、可部署；站在河流上攻击力 -1，被攻击时受到伤害 +1，攻击奕星师伤害 -1 |
| 高地 | 射手 / 法师射程 +1 |

MVP 注意：河流允许部署仅在己方部署区内生效；公共区河流不能被直接部署。

## 9. 策略卡规则

MVP 阶段策略卡可先实现简化版本。

| 策略卡 | 修订后效果 |
|---|---|
| 火矢 | 所有射手本回合攻击力 +2 |
| 鼓舞 | 所有己方武将本回合攻击力 +2 |
| 落石 | 敌方半场随机 3 格各造成 5 伤害 |
| 补给 | 己方奕星师回复 10 HP，不超过最大 HP |
| 急行军 | 己方全军本回合移动 +1 |
| 地震 | 双方奕星师各受到 10 伤害 |

备注：技能数值与策略卡数值仍在规划中，当前数值用于原型验证，不代表最终上线数值。

## 10. 战斗内查看

对战中点击棋盘单位可打开详情浮层，不打断战斗流程。

详情展示：

- 武将名。
- 品质。
- 职业。
- 当前生命值 / 生命上限。
- 攻击力。
- 射程。
- 移动。
- 物理格挡 / 法术格挡。
- 伤害类型。
- 技能说明。
- 等级。
- 当前异常状态与增益状态。

展示数据必须来自当前战斗实例，包含等级、光环、地形、策略卡等实时修正后的结果。

召唤物也应可查看，至少展示：来源、基础属性、行动逻辑、是否可被技能影响。

## 11. 战斗结算与统计

### 11.1 MVP 结算信息

结算页至少展示：

- 胜负。
- 总回合数。
- 双方奕星师剩余 HP。
- 存活单位数。
- 铜钱奖励。
- 经验 / 天梯 / 券类奖励可先占位。

### 11.2 本场战斗数值统计

先保留本场战斗数值统计，不强制选定 MVP 武将。

建议记录：

- 斩将数量。
- 对敌方奕星师造成的总伤害。
- 对敌方单位造成的总伤害。
- 承受伤害。
- 治疗量。
- 召唤单位数量。
- 控制次数。
- 存活回合数。

### 11.3 MVP 武将评选

MVP 武将评选暂缓开发。

如果后续需要，可基于多维评分：

```text
MVP 分 = 斩将数权重 + 奕星师伤害权重 + 单位伤害权重 + 抗伤权重 + 治疗权重 + 控制权重 + 存活权重
```
