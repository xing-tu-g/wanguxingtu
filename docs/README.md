# 万古星图：开发文档索引

此目录用于承接原始 GDD，并拆成 Hermes + Codex + Godot 可执行的开发输入。

## 文档列表

1. `01_rules_spec.md`  
   核心规则规格书。定义奕星师、棋盘、部署区、回合制流程、移动、攻击、地形、策略卡、战斗统计等。

2. `02_values_and_content.md`  
   数值与内容表。集中管理职业模型、奕星师等级、武将升级、42 人武将池、MVP 12 武将、地形、策略卡、奖励和战斗统计字段。

3. `03_godot_mvp_plan.md`  
   Godot MVP 开发清单。包含工程结构、里程碑、页面优先级、安卓横屏 UI 规范和 Codex 执行建议。

4. `04_battle_details_data_tests.md`  
   战斗细则、数据结构与测试清单。定义单位行动顺序、移动/攻击细则、技能效果持续、召唤物、手牌刷新、胜负边界、JSON 配置、存档结构和测试用例。

5. `05_battle_ui_layout_spec.md`  
   战斗界面对弈式布局设计稿。定义“平面星图棋盘 + 左右奕星师对弈 + Q版武将小人战斗表现”的横屏构图、尺寸比例、常驻信息、子窗口和美术出图说明。

6. `06_art_asset_spec.md`  
   Q版星图战棋美术资产规格。定义 Q版武将小人、动作反馈、技能特效、棋盘、UI、奕星师、背景资产尺寸、命名、导出格式和 Godot 接入路径。

7. `HANDOFF.md`  
   长会话阶段交接记录。每完成一个阶段后写入真实验证结果、产物路径、阻塞项和下一步，供后续会话恢复项目状态。

## 当前设计决议

- “弈天师”统一改名为“奕星师”。
- 战斗初始 HP = 奕星师等级生命值。
- 奕星师经验仅能通过升级武将等级获得。
- 补给回复 10 HP。
- 地震造成双方奕星师各 10 伤害。
- 战斗是回合制自动战棋，不是即时战斗。
- 双方各自靠近己方奕星师的前三列为部署区，中间四列为公共区域。
- 攻击目标优先选择攻击范围内“距离我方阵地最近”的敌方单位。
- 单位行动顺序为前方单位先行动，后方单位后行动。
- 单位先判定攻击；若攻击范围内有敌方单位则直接攻击，否则向前移动后再次判定攻击。
- 攻击距离统一使用曼哈顿距离。
- 每个格子仅能存在一个单位，不允许斜向移动，MVP 不做绕路。
- 同一效果不可重复，所有临时效果 MVP 阶段均只持续一回合。
- 阵容同名限制：传说 1 张、史诗 2 张、精英 3 张。
- 每局开始抽 3 张牌，手牌上限 5 张，允许锁定卡牌不被刷新。
- 双方同时 HP <= 0 时，判定双方均失败。
- 抽卡系统 MVP 阶段占位，暂不开发。
- MVP 武将评选暂缓，只保留本场战斗数值统计。
- 美术规范改为横屏安卓手机适配规范，不再使用 Word 文档的 cm / pt 作为实现单位。
- 战斗美术方向改为“英气 Q 版星图战棋”：Q版武将小人、动作反馈和技能特效是爽感核心；背景只做开阔低干扰舞台。
